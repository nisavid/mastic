# Domain docs

This is a single-context repository.

Before exploring a task, read root `CONTEXT.md` and any relevant system-wide
ADRs under `docs/adr/`. If either is absent, proceed silently.

Use the glossary's canonical terms in issues, proposals, tests,
documentation, and code. Do not substitute words listed under `_Avoid_`.

When a required concept is missing, reconsider whether it belongs to the
domain. If it does, use `domain-modeling` to resolve and record it.

Surface contradictions with an existing ADR explicitly instead of silently
overriding it.
