# Security Policy

## Supported versions

MASTIC does not have a released runtime yet. Security fixes apply to the
default branch and its repository automation. Historical imported components
remain available through Git history and are not supported as separate
products.

## Reporting a vulnerability

Report suspected vulnerabilities through
[GitHub private vulnerability reporting](https://github.com/nisavid/mastic/security/advisories/new).
Do not open a public issue, pull request, or discussion for an undisclosed
vulnerability.

Include the affected component and revision, the expected impact, reproduction
steps or a proof of concept, and any known mitigations. Reports should avoid
real credentials, private data, and destructive testing.

We will acknowledge a report within 7 days and provide a status update at
least every 14 days until it is resolved or declined. We will coordinate the
disclosure timeline with the reporter and ask that details remain private until
a fix or advisory is available.
